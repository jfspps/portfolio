CloudWrite

Overview

CloudWrite is an article planning/authoring tool designed particularly for science writers. Multiple users can login and edit each article making CloudWrite a collaborative tool. A REST API web service is offered and can publish either XML or JSON feeds. Many of the technologies (e.g. H2 and MySQL databases, BCrypt, CSRF and timestamps) learned when building Secure-SRM are also applied here.

A compiled JAR file is provided here for demonstration purposes.

Motivation

Currently, I am a freelance science writer and also maintain LearningChem where I publish articles for students and teachers. Following on from my experience of building Secure-SRM, I decided to build an app which would help me plan new articles. I also wanted to revisit Spring development and get some experience with building a REST API web service.

Landing page

Users authenticate their identity and are then granted equal read/write authorisation roles for all articles on file.

Admin users

Admin users of CloudWrite are granted roles which permit them to manage user accounts. They can also edit articles just as regular users do.

User admin page

Users can update their own password.

Research and fundamental pieces

An research piece/article highlights an area of research. A fundamental piece is an article which explains key concept(s). The user pages for research pieces (below) and fundamental pieces are similar.

Users can create new research pieces (below) and fundamental pieces.

REST API

The REST API web service is documented using OpenAPI (formerly Swagger). A shortcut to the OpenAPI documentation page is provided at the login page. All users, authenticated or not, are permitted read-only access to all research and fundamental pieces. In addition, users can search by keyword to narrow down the returns.
